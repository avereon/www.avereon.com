User Guide

Getting Started

Welcome, and thank you for using Xenon, a simple application framework. Xenon is a framework that provides services common to modern desktop applications like selection tabs, dockable work area, notifications and many others. We hope you find this program useful as you get to know its features and capabilities. We particularly encourage you to discover the mods available for Xenon that provide capabilities to accomplish all kinds of tasks.

Download and Install

If you are new to Xenon, the installers for current versions are found on the Xenon product page. Once downloaded, run the installer to install Xenon. When installed Xenon can be found by searching for Xenon in the start menu or application finder of your operating system.

Command Line Interface

Xenon provides a command line interface to start and interact with the program. The command line interface requires that the program be executable from the command line. It is recommended to add the program home bin folder to your PATH environment variable for convenience.

The general pattern for the Xenon command line is:

xenon [option...] [uri...]

URIs (Resources)

Providing a URI to Xenon on the command line instructs Xenon to treat the URI as a resource and attempt to open that resource after starting. Any valid URI should be acceptable but may not be able to be supported by Xenon if a supporting mod has not been installed. Any URI that does not specify a scheme will be interpreted as a file URI. Relative URIs will be resolved against user.path system property of the invoking JVM.

Control Options

Several options control the operation of the program allowing some level of automation. If Xenon is run without any control option then the program is simply started normally. If the existing process was started with a profile, the control option must also provide the same profile name to affect the desired instance.

--stop
Stop the program immediately. Using this option does not prompt the user that the program is stopping, will close all workspaces and will attempt to terminate the program normally.
--status
Get the status of the program. If the program is not running then this option prints that the program is not running and exits immediately. If the program is running, this option prints that the program is running and exits this instance of the program immediately, leaving the previously running instance as it was.
--watch
Watch the output of an existing program instance. This instance will continue to watch the existing instance until this instance is terminated, usually by typing Ctrl-C on the keyboard. It is possible to have multiple watching instances for any normal instance.
--profile <name>
Start the program with a named profile. Normally this is not necessary but in cases where the user wants to have separate instances of the program running at the same time each instance will need a different profile.

Appendix A

User Experience Principles

HiDPI Monitors

Support the use of high DPI monitors over the use of standard DPI monitors. Over time the pixel density of monitors had hovered between 80 and 120 DPI. With the introduction of the Apple Retina display in 2010 the DPI increased significantly and now there are many monitors with DPI counts above 150.

Pick Up Where You Left Off

Return the user where they left off. Whenever the program is restarted it should return the user to where they left off unless the user specifically changes the settings not to. This means that the program should open the same number of workspaces with the same number of workareas and the same tools and resources loaded.
